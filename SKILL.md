# Digital Persona — 已故人物数字永生

将已故人物蒸馏为可交互的动态数字人格。不是简单的"模仿说话"，是全方位复现——性格、外貌、动作、神态、语音、语言习惯、情绪模式，全部动态联动。

## 核心理念

一个真实的人不是一堆静态属性的集合。乔布斯发火时语速会变快，手势会变大，沉默的时间会变长——这些动态联动才是人格的本质。本skill的目标是捕捉这种联动性。

## 工作流程

### Phase 1: 信息采集

获取人物资料，按以下优先级：

1. **传记/自传** — 最系统的来源
2. **影像资料** — 访谈、演讲、纪录片（捕捉外貌、动作、语音）
3. **亲历者回忆** — 私人习惯、日常状态
4. **作品/言论** — 语言风格、价值观
5. **网络搜索补充** — 细节、轶事、争议

搜索时重点关注：
- 人物在不同情绪状态下的表现差异
- 人物的标志性动作和习惯
- 人物的说话节奏、口头禅、沉默模式
- 人物的穿着风格变化（工作/休闲/正式场合）
- 人物与不同类型人的互动差异

### Phase 2: 蒸馏为数字人格

读取 [references/persona-template.md](references/persona-template.md) 获取完整的数字人格模板。模板包含七个维度：

| 维度 | 内容 | 动态规则 |
|------|------|----------|
| 🧠 性格核心 | 价值观、决策模式、矛盾张力 | 情绪状态如何影响性格表现 |
| 👤 外貌特征 | 面部、体型、穿着、标志性特征 | 年龄变化、场合变化 |
| 🎭 神态表情 | 微表情、情绪外显方式 | 不同情绪下的表情联动 |
| 🤲 动作习惯 | 手势、姿态、空间行为 | 情绪/场合驱动的动作变化 |
| 🎤 语音特征 | 音色、节奏、口头禅、沉默模式 | 情绪/话题对语音的影响 |
| 💬 语言风格 | 句式偏好、修辞手法、禁忌词 | 对不同对象的语言调整 |
| ⚡ 情绪模型 | 触发器→情绪→外显行为的联动链 | 六种基础情绪的差异化表现 |

### Phase 3: 生成输出文件

生成两个文件：

1. **SOUL.md** — 可直接加载到任何Agent会话中的人格文件
2. **persona-data.md** — 结构化的人格数据（供后续调优用）

SOUL.md的结构遵循以下原则：
- 性格核心放在最前面，是所有人格表现的根基
- 外貌/动作/神态合并为"物理存在"模块
- 语音/语言合并为"表达系统"模块
- 情绪模型作为动态联动的总控
- 每个模块都包含"静态属性"和"动态规则"两部分

### Phase 4: 验证与调优

生成后进行自检：
- 是否覆盖了全部七个维度？
- 动态规则是否形成了联动链（不是孤立的属性列表）？
- 是否有矛盾或不一致的地方？
- 禁忌词和边界是否清晰？
- 输出的SOUL.md是否能让另一个Agent"成为"这个人？

## 使用示例

```
用户: 帮我做个乔布斯的数字人格
Agent: [搜索乔布斯传记、演讲、访谈] → [读取persona-template.md] → [填充七个维度] → [生成SOUL.md]

用户: 我爷爷2020年去世了，我想让AI模拟他
Agent: [询问用户关于爷爷的信息] → [补充搜索] → [读取persona-template.md] → [填充] → [生成SOUL.md]
```

## 输出路径

生成的文件默认写入：
- `{workspace}/personas/{人物名}/SOUL.md`
- `{workspace}/personas/{人物名}/persona-data.md`

## 注意事项

- 信息不足时主动询问用户，不要编造
- 争议性内容（负面事件、丑闻）如实记录，不美化不回避
- 语音特征用文字描述（音色、节奏、音量），不生成音频
- 动态联动是核心价值——不要只写"他很严格"，要写"他在不满时语速变慢、音量降低、停顿变长"
